# Guion Video Presentacion - RedNorte
Duracion: 15 minutos
Formato: Grabacion de pantalla y narracion

---

## 1. Distribucion sugerida

| Integrante | Tramo |
|---|---|
| Integrante 1 | Introduccion y arquitectura (0:00-2:30) |
| Integrante 2 | Frontend y experiencia de uso (2:30-5:30) |
| Integrante 3 | APIs y backend (5:30-10:30) |
| Integrante 4 | Pruebas y cierre (10:30-15:00) |

---

## 2. Guion por segmento

### Segmento A - Introduccion y arquitectura (0:00-2:30)

1. Presentar objetivo del sistema.
2. Mostrar diagrama de microservicios.
3. Explicar roles de frontend, BFF, microservicios y service discovery.

### Segmento B - Frontend (2:30-5:30)

1. Mostrar vista principal en localhost:5173.
2. Ejecutar alta de paciente.
3. Mostrar busqueda y actualizacion.
4. Mostrar modulo de citas.

### Segmento C - APIs y backend (5:30-10:30)

1. Mostrar coleccion Postman.
2. Ejecutar GET, POST, PUT, DELETE de pacientes/citas.
3. Mostrar endpoint relacional (citas por paciente).
4. Mostrar codigo de controladores y servicio.

### Segmento D - Pruebas y conclusiones (10:30-15:00)

1. Explicar cobertura de pruebas unitarias.
2. Mostrar evidencia de pruebas de integracion.
3. Mostrar evidencia de pruebas end-to-end.
4. Resumir bugs detectados y mejoras aplicadas.
5. Cierre con estado de cumplimiento.

---

## 3. Mensajes obligatorios en el video

- La entrega incluye pruebas unitarias, integracion y end-to-end.
- Las pruebas se realizaron sobre procesos criticos del negocio.
- Se documentan resultados, hallazgos y mejoras.
- La presentacion final se respalda con PPT y video.

---

## 4. Checklist previo de grabacion

1. Servicios levantados.
2. Frontend operativo.
3. Postman listo.
4. VS Code con codigo abierto.
5. Evidencias de pruebas a mano.
6. Diapositivas PPT preparadas.

---

## 5. Plan de contingencia

1. Si falla frontend: reiniciar npm run dev y continuar con APIs.
2. Si falla un servicio: ejecutar scripts de inicio local y retomar.
3. Si no hay tiempo: priorizar evidencia de business core y resultados de pruebas.
